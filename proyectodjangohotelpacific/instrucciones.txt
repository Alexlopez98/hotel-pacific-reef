GUÍA DE EJECUCIÓN - PROYECTO HOTEL PACIFIC REEF
================================================

Este documento contiene los pasos detallados para configurar y ejecutar el proyecto 
'proyectodjangohotelpacific' conectándolo correctamente a la base de datos Oracle Cloud 
y al almacenamiento de imágenes en Amazon S3.

1. REQUISITOS PREVIOS
---------------------
* Python 3.10 o superior instalado.
* Git instalado y configurado.
* El archivo Wallet (.zip) de la base de datos 'Ingenieria_Software' de Oracle Cloud.
* El cliente de Oracle (librería oracledb instalada en el entorno virtual).

2. CONFIGURACIÓN INICIAL DEL PROYECTO
-------------------------------------
1. Clonar el repositorio desde GitHub:
   git clone <URL_DEL_REPOSITORIO>
   cd hotel-pacific-reef/proyectodjangohotelpacific

2. Crear y activar el entorno virtual (venv):
   python -m venv venv
   .\venv\Scripts\activate   (En Windows)

3. Instalar las dependencias:
   pip install -r requirements.txt

3. CONFIGURACIÓN DE LA BASE DE DATOS (ORACLE CLOUD)
---------------------------------------------------
CADA DESARROLLADOR DEBE CONFIGURAR SU RUTA LOCAL. No subir estos cambios a GitHub.

1. Descomprimir el Wallet en una carpeta segura (ej: C:/Wallet_IngenieriaSoftware).
2. Abrir el archivo 'hotel/settings.py'.
3. Localizar el bloque DATABASES y modificar los siguientes campos:

   'OPTIONS': {
       'config_dir': 'C:/RUTA_A_SU_WALLET',
       'wallet_location': 'C:/RUTA_A_SU_WALLET',
       'wallet_password': 'Su_Contraseña_Del_Wallet',
   }

*IMPORTANTE: En Windows, use barras normales (/) en la ruta para evitar errores.*

4. CONFIGURACIÓN DE IMÁGENES (AMAZON S3)
----------------------------------------
Por seguridad, las llaves de acceso no están en GitHub. Para que las imágenes carguen:

1. Solicite las llaves de AWS al administrador (Alexander).
2. En 'hotel/settings.py', busque la sección de Amazon S3 y reemplace:
   
   Amazon_ACCESS_KEY_ID = 'AKIAU73ZW2LFQSFRTHAX'
   Amazon_SECRET_ACCESS_KEY = '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'

5. EJECUCIÓN DEL SERVIDOR
-------------------------
Con el entorno virtual activo y en la carpeta raíz del proyecto, ejecute:

python manage.py runserver

El servidor debería iniciar en: http://127.0.0.1:8000/

6. SOLUCIÓN DE PROBLEMAS FRECUENTES
-----------------------------------
* Error "Enter PEM pass phrase": Verifique la 'wallet_password' en settings.py.
* Error "DPY-4000": El nombre 'ingenieriasoftware_low' debe coincidir con su tnsnames.ora.
* Carga infinita: Verifique que su dirección IP actual esté autorizada en la ACL de Oracle Cloud (Consola de Red).
