GUÍA DE EJECUCIÓN - PROYECTO HOTEL PACIFIC REEF
================================================

Este documento contiene los pasos detallados para configurar y ejecutar el proyecto 
'proyectodjangohotelpacific' conectándolo correctamente a la base de datos Oracle Cloud 
y al almacenamiento de imágenes en Amazon S3 de forma segura.

1. REQUISITOS PREVIOS
---------------------
* Python 3.10 o superior instalado.
* Git instalado y configurado.
* El archivo Wallet (.zip) de la base de datos 'Ingenieria_Software' de Oracle Cloud.
* El cliente de Oracle (librería oracledb instalada en el entorno virtual).

2. CONFIGURACIÓN INICIAL DEL PROYECTO
-------------------------------------
1. Clonar el repositorio desde GitHub:
   git clone <URL_DEL_REPOSITORIO>
   cd hotel-pacific-reef/proyectodjangohotelpacific

2. Crear y activar el entorno virtual (venv):
   python -m venv venv
   .\venv\Scripts\activate   (En Windows)

3. Instalar las dependencias (Asegúrese de que el venv esté activo):
   pip install -r requirements.txt

3. CONFIGURACIÓN DE SEGURIDAD (ARCHIVO .ENV)
--------------------------------------------
Para evitar bloqueos de seguridad en GitHub y proteger las cuentas, las llaves de AWS 
se manejan de forma local.

1. En la raíz del proyecto (donde está el archivo manage.py), cree un nuevo archivo 
   llamado exactamente: .env
2. Abra el archivo .env con el Bloc de Notas y pegue lo siguiente:

   AWS_ACCESS_KEY_ID=SOLICITAR_KEY_AL_ADMIN
   AWS_SECRET_ACCESS_KEY=SOLICITAR_SECRET_AL_ADMIN

3. Guarde y cierre el archivo. (Este archivo está en el .gitignore y no se subirá al repo).

4. CONFIGURACIÓN DE LA BASE DE DATOS (ORACLE CLOUD)
---------------------------------------------------
Cada desarrollador debe configurar su ruta local en 'hotel/settings.py':

1. Descomprimir el Wallet en una carpeta segura (ej: C:/Wallet_IngenieriaSoftware) la cual esta en onedrive compartido.
2. Localizar el bloque DATABASES y modificar los siguientes campos:

   'OPTIONS': {
       'config_dir': 'C:/RUTA_A_SU_WALLET',
       'wallet_location': 'C:/RUTA_A_SU_WALLET',
       'wallet_password': 'Su_Contraseña_Del_Wallet',
   }

*IMPORTANTE: Use barras normales (/) en las rutas para evitar errores en Windows.*

5. EJECUCIÓN DEL SERVIDOR
-------------------------
Con el entorno virtual activo, ejecute:

python manage.py runserver

El servidor debería iniciar correctamente en: http://127.0.0.1:8000/

6. SOLUCIÓN DE PROBLEMAS FRECUENTES
-----------------------------------
* Error "Pillow is not installed": Ejecute 'pip install Pillow' nuevamente.
* Error "Enter PEM pass phrase": Verifique la 'wallet_password' en settings.py.
* Error "DPY-4000": El nombre 'ingenieriasoftware_low' debe coincidir con su tnsnames.ora.
* Imágenes no cargan: Verifique que el archivo .env tenga las llaves de AWS correctas.
